# Capstone demo day

**Length:** 60–90 min (scales with cohort) · **Format:** live, everyone presents

## Purpose
Each participant demos one real recurring task they automated with Claude Code (skill, agent,
hook, or MCP). This is where the program proves its ROI and where the demo bank grows.

## Format
- **5 min per person:** the task before, what they built, a live run, time saved.
- Keep it live — running the thing beats slides.
- Capture each one: a short README + the artifact go into `demos/` (or the shared skills repo).

## What "good" looks like
- It runs end-to-end without hand-holding.
- It's something they actually do repeatedly (not a toy).
- They can state the time saved ("this was 2 hrs/week of copy-paste").

## Scoring / feedback (light)
| Dimension | Prompt |
|-----------|--------|
| Real | Is this a task you actually repeat? |
| Works | Did it run clean, live? |
| Reusable | Could a teammate use it tomorrow? |
| Saved | Rough time saved per week? |

## After demo day
- Roll up the metrics for leadership: total artifacts shipped, median time saved, 30-day active use.
- Merge the best capstones into the shared library so the next cohort starts ahead.
- Schedule recurring office hours — adoption happens between cohorts, not during them.

## Facilitator prep
- Confirm everyone can screen-share and run live.
- Have the [operating-model](../operating-model.md) metrics sheet open to log results.
